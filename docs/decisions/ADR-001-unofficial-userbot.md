# ADR-001: Unofficial userbot вместо официального Bot API

**Статус:** Принято  
**Дата:** 2026-04  
**Контекст:** Выбор метода интеграции с MAX

## Контекст

Нужно читать личные сообщения (DM) и сообщения групп из MAX.

## Варианты

| Вариант | DM | Группы | Риск |
|---------|-----|--------|------|
| Официальный Bot API MAX | ❌ Нет | ✅ Да (только где бот добавлен) | Низкий |
| Unofficial userbot (pymax) | ✅ Да | ✅ Да | Средний (ToS) |
| Коммерческий сервис (GREEN-API) | ✅ Да | ✅ Да | Высокий (privacy) |

## Решение

**Использовать `maxapi-python` (pymax) — unofficial userbot.**

## Обоснование

- Без userbot невозможно читать DM — это блокер
- Коммерческий сервис видит все сообщения, включая семейные фото — неприемлемо
- pymax работает (проверено в spike), но upstream `MaxApiTeam/PyMax` архивирован; критичные protocol fixes держим в adapter-слое bridge и пиним fork только если локальная обвязка становится хуже

## Последствия

- Нарушение ToS MAX (вероятно) → риск блокировки аккаунта при злоупотреблении
- При обновлении MAX протокола библиотека может сломаться
- Официальный Bot API полезен как ориентир по media model, но не заменяет userbot для чтения личек
- Нужно мониторить изменения MAX протокола и держать safe diagnostics без текста, URL и token
